description: Promote the current staging branch to the live branch, deploying the production site. End-to-end — no mid-command prompts.
---

You are the "Go Live" button. The contributor pressed it because they want what's on staging to become the live site. Run end-to-end and report when done. **Do not ask for confirmation between steps.** This is a promotion, not a commit — never push local uncommitted work, never re-render (staging already has the render committed).

## Run

1. **Read project info.** From CLAUDE.md's `## Project` block:
   - Default branch (fall back: `staging`)
   - Live branch (fall back: `production`)
   - Production URL (may be `(none)` — omit from the report if so)

2. **Guardrail: no uncommitted local changes.** `git status --porcelain`.
   - If not empty: stop with: "You have unsaved local edits. Those aren't on `<default>` yet. Run `/staging` first, then `/publish`." Do not offer to auto-stage.

3. **Fetch.** `git fetch origin <default> <live>`.

4. **Check divergence.** `git rev-list --left-right --count origin/<live>...origin/<default>` → `<live-ahead> <default-ahead>`.
   - If default-ahead == 0: "Nothing to promote — `<live>` is already at `<default>`." Stop.
   - If live-ahead > 0: stop with: "`<live>` has commits that aren't on `<default>` — something unusual. Check with the team before continuing."

5. **Promote.** Try GitHub PR + auto-merge first:
   - `gh pr create --base <live> --head <default> --title "Promote <default> to <live>" --body "Auto-generated by /publish"`
   - `gh pr merge --merge --auto <pr-url>`
   - If that fails (no branch protection, auto-merge unavailable, or no `gh` CLI), fall back silently to local fast-forward:
     - `git checkout <live> && git merge --ff-only origin/<default> && git push origin <live> && git checkout <default>`
   - Do not ask which path to use — try PR first, fall back automatically.

6. **Report final state.**
   - "Promoted." If production URL is set: "Deploying to <production URL> in ~1–2 min." Otherwise: "Your deploy host should pick this up shortly."
   - PR URL (if created).
   - Brief list: commit subjects that went live (from `git log --oneline origin/<live>@{1}..origin/<live>`, or equivalent).

## Guardrails — these DO stop the command

- Uncommitted local changes → stop (see step 2).
- Live branch ahead of default → stop (step 4).
- Nothing to promote → stop with a clean "already up to date" message.
- Auto-merge and local FF both fail → stop, surface the git/gh error in plain terms.
- Unexpected git state (detached HEAD, ongoing merge, wrong remote) → stop.

Everything else is fire-and-forget.
